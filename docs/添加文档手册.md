# 添加文档操作手册

每加一份新文档,照着下面 5 步做。`<占位符>` 的地方按实际情况替换。

## 通用流程

**1. 下载 PDF**
把文档存成 PDF,放进 `data/raw/<名字>.pdf`。
- USCIS / 政府网页:浏览器 `Cmd+P` → 存储为 PDF
- 本身就是 PDF 的(如表格说明):直接下载
记下它的来源网址。

**2. 登记来源**
在 `data/sources.md` 里加一行:文件名、网址、下载日期、说明。

**3. 解析**
```bash
docker compose exec app python -m src.ingestion.parse_pdf \
    data/raw/<名字>.pdf \
    --output data/processed/<名字>.jsonl
```
✓ 看到 `Parsed XX blocks`。
（注:非 USCIS Policy Manual 的文档，章节标题可能识别不出来，会提示 "no section headings detected"——没关系，分块照常工作，只是引用会粗一点。）

**4. 嵌入入库**
```bash
docker compose exec app python -m src.ingestion.embed_and_index \
    --jsonl data/processed/<名字>.jsonl \
    --prefix "<引用前缀>" \
    --source-url "<来源网址>" \
    --title "<文档标题>" \
    --publisher <USCIS / SEVP / DOS / NAFSA / OISS> \
    --tier <1=权威官方 / 2=实务参考> \
    --source-pdf data/raw/<名字>.pdf
```
✓ 看到 `Inserted XX chunks for document_id=N`。

**5. 验证**
```bash
docker compose exec postgres psql -U postgres -d postgres -c "SELECT COUNT(*) FROM visa.chunks;"
```
chunk 总数应该变大了。再用一个相关问题测一下:
```bash
docker compose exec app python -m src.generation.rag "<跟新文档相关的问题>"
```

## 推荐接下来加的文档

| 文档 | 从哪拿 | publisher | tier | 建议 prefix |
|---|---|---|---|---|
| Form I-765 说明 | uscis.gov/i-765 → "Instructions for Form I-765 (PDF)"，本身是 PDF | USCIS | 1 | `USCIS Form I-765 Instructions` |
| STEM OPT Hub | studyinthestates.dhs.gov/stem-opt-hub（网页转 PDF，可能要存几页） | SEVP | 1 | `DHS STEM OPT Hub` |
| 8 CFR §214.2(f) | ecfr.gov 上搜 "8 CFR 214.2"，有下载按钮 | eCFR | 1 | `8 CFR 214.2(f)` |
| USCIS Policy Manual 其他章节（如 Maintaining Status） | uscis.gov/policy-manual/volume-2-part-f | USCIS | 1 | `USCIS Policy Manual, Vol 2, Part F, Ch X` |

建议顺序:先加 **Form I-765 说明**(本身是 PDF,最省事),再加 **STEM OPT Hub**(和你自己最相关),其余有精力再加。

## 小提示

- 每份文档的 `--source-url` 用它**实际的网址**,不要都填同一个。
- `--prefix` 会变成引用路径的一部分,写得清楚一点(用户看引用时能看懂)。
- 加完 3–4 份文档后,知识库就够厚了,可以进入下一阶段(评估集、时间线计算器)。
- 每加完一两份,记得 `git add . && git commit -m "..." && git push`。
